# CEO Plan: Data Anonymizer v2.0 вҖ” Image Anonymization + Cross-Platform GUI Distribution

Generated by /plan-ceo-review on 2026-03-26
Branch: main | Mode: SCOPE EXPANSION

## Vision

### 10x Check

Transform the data-anonymizer from a personal Claude Code hook into a **campus data security gateway** вҖ” a standalone Windows application that any school staff member can use to sanitize files before sending to any AI tool. Full media coverage (text + images), two compliance modes (reversible/irreversible), drag-and-drop simplicity.

### Platonic Ideal

A Windows installer that a non-technical school administrator downloads, runs, and immediately starts using:
- Drag files/folders in вҶ’ see a Before/After preview вҶ’ click "Export"
- Choose "Pseudonymization" (reversible, for personal use) or "Anonymization" (irreversible, for sharing/compliance)
- Images: school logos blurred, faces blurred, text PII (school names, personal names, IDs) black-boxed
- Text: all PII replaced with tokens or generic labels
- Config import: download a .zip from the IT admin вҶ’ one-click setup with all school-specific terms and logo templates
- Auto-update notification: "v1.2 available, click to download" (passive only, no auto-install)
- The user feeling: "Oh, this is so simple вҖ” just drag and click."

### Key Constraints
- **Target platform:** Windows (colleagues' machines) вҖ” dev on macOS, build via GitHub Actions
- **AI-agnostic:** This is a pre-processing tool, not tied to any specific AI service
- **No Python on target:** Must be a standalone distribution via PyInstaller
- **Two anonymization modes:** Reversible (pseudonymization) + Irreversible (anonymization, еҸ°зҒЈеҖӢиіҮжі• compliant)
- **Large dependencies:** PyTorch + ckip-transformers for Full build; Lite build without NER for smaller binary
- **Distribution format:** PyInstaller `--onedir` (not `--onefile`) because Tesseract binaries must be bundled alongside, wrapped in an Inno Setup installer for clean UX
- **Network:** School networks may block GitHub; auto-update check must be passive and timeout-safe (3s timeout, silent failure)

## Accepted Scope (added to this plan)

- Irreversible anonymization mode (`reversible=False` вҶ’ generic labels like `[дәәеҗҚ]`/`[еӯёж Ў]`, no mapping file produced, existing mapping files ignored)
- Image anonymization pipeline:
  - OCR text PII: Tesseract (via Presidio Image Redactor) вҶ’ existing detectors (custom/regex/NER)
  - Face detection: OpenCV DNN with `res10_300x300_ssd_iter_140000.caffemodel` (~10MB, CPU-only, no GPU needed)
  - Logo detection: OpenCV `matchTemplate` with multi-scale search (user provides logo PNG in config)
  - Known limitation: template matching does not handle rotated/skewed logos (acceptable for v2; deferred to v3)
- Each image pipeline stage is independent вҖ” if one fails (e.g., Tesseract not found), it logs a warning and continues with remaining stages
- **Reversibility semantics for images:** Image obfuscation (blur/black-fill) is **always visually irreversible** regardless of mode. The `reversible` flag only affects **text PII**: reversible mode uses tokens (`__ANON:SCHOOL_001__`) with mapping file; irreversible mode uses generic labels (`[еӯёж Ў]`) with no mapping. Face/logo blur cannot be "undone" вҖ” this is by design and clearly communicated to the user.
- **Image output format:** Output images retain their original format (PNGвҶ’PNG, JPGвҶ’JPG, etc.). **Exception:** JPEG outputs use quality=95 to minimize compression artifacts around redacted regions. For irreversible mode with black fill, JPEG is acceptable because the fill is solid.
- **PDF image handling:** PDFs containing rasterized images are converted page-by-page to images using `pdf2image` (poppler backend). Each page is processed through the full image pipeline. All pages are processed in batch mode, not just the first. Text-layer PDFs continue to use the existing text pipeline via pdfplumber.
- **DOCX/PPTX embedded images:** Out of scope for v2. Only standalone image files (.jpg/.png/.bmp/.tiff) are processed by the image pipeline. Embedded images in Office documents are a v3 feature.
- Windows GUI with tkinter: file dialog button (primary) + TkDND drag-and-drop (best-effort)
  - **Drag-and-drop strategy:** Use file dialog button as the primary and always-available input method. Attempt to load TkDND for native drag-and-drop; if TkDND DLL is unavailable or fails to load, silently fall back to file dialog only. The drop-zone area doubles as a large "Click to select files" button.
  - Preview: max 800px wide thumbnail, single file at a time; for multi-page PDFs show first page only in preview (all pages are still processed)
  - Text preview: diff-highlighted view showing replaced spans in color
  - Image preview: side-by-side with red boxes on original, blurred regions on result
- Batch processing: folder input, recursive scan, output to `{original_folder}_anonymized/` preserving directory structure
  - **Batch mode skips interactive preview** вҖ” processes all files directly, shows summary report at the end. User can review individual output files afterward by clicking them in the results list.
- GitHub Actions CI/CD: push tag вҶ’ Windows runner вҶ’ PyInstaller `--onedir` вҶ’ Inno Setup installer вҶ’ upload to GitHub Releases
  - **Disk space:** Full build requires ~10GB free disk; CI workflow includes a disk-cleanup step (`actions/runner` cleanup of pre-installed tools). If `windows-latest` proves insufficient, fall back to self-hosted runner.
- Auto-update check: fires in a **background thread** immediately after the main window renders; result posted to main thread via tkinter `after()` callback. HTTP GET to `https://api.github.com/repos/Imbad0202/data-anonymizer/releases/latest` with 3s timeout; on failure, silently skip; on success, show banner "v{new} available" with link to Releases page in default browser; no auto-download. **Network note:** school/corporate firewalls may block GitHub API; this is expected and handled by the silent timeout.
- Config export/import:
  - Export: `.anonymizer-config.zip` containing `config.json` + `logo_templates/` directory
  - Import: GUI "Import Config" button вҶ’ select .zip вҶ’ extract to app data dir вҶ’ validate schema version
  - Config schema v1:
    ```json
    {
      "version": 1,
      "custom_terms": {"schools": ["ж ЎеҗҚ"], "people": ["дәәеҗҚ"]},
      "file_types": [".txt", ".md", ".docx", ".pdf", ".jpg", ".png"],
      "logo_templates": ["ntu_logo.png"],
      "substring_match": true
    }
    ```
    - `custom_terms`: dict of category вҶ’ list of sensitive terms (used by CustomDetector)
    - `file_types`: list of file extensions to process
    - `logo_templates`: list of logo image filenames in the `logo_templates/` directory
    - `substring_match`: if true, match terms as substrings (e.g., "еҸ°зҒЈеӨ§еӯё" matches "еңӢз«ӢеҸ°зҒЈеӨ§еӯёиіҮиЁҠзі»")
    - Note: `scan_paths` is only used in CLI/Hook mode (Claude Code); GUI mode uses drag-and-drop input instead
  - First-launch detection: if no config exists and a `.anonymizer-config.zip` is in the same directory as the exe, auto-prompt to import
- **Tesseract bundling:** Bundle portable Tesseract binary + `tessdata/` with language packs `eng` + `chi_tra` (Traditional Chinese, ~23MB) in `_internal/tesseract/`. Set `TESSDATA_PREFIX` environment variable at launcher startup pointing to `_internal/tesseract/tessdata/`. This is critical for Taiwanese school documents with Chinese-character PII.
- Lite mode: separate build target without PyTorch/ckip-transformers; only custom terms + regex detectors; estimated ~150-250MB (hard limit 300MB, pending actual measurement after first build)
- Full mode: includes NER; estimated ~2-3GB; same GitHub Actions workflow, separate release asset

## Image Pipeline Detail

```
  INPUT IMAGE
       в”Ӯ
       в”ңв”Җв”Җв–¶ [Stage 1: OCR Text PII]
       в”Ӯ     Tesseract OCR вҶ’ extract text + bounding boxes
       в”Ӯ     вҶ’ run existing detectors (custom/regex/NER)
       в”Ӯ     вҶ’ collect PII regions with pixel coordinates
       в”Ӯ     Failure mode: Tesseract not found вҶ’ log warning, skip to Stage 2
       в”Ӯ
       в”ңв”Җв”Җв–¶ [Stage 2: Face Detection]
       в”Ӯ     OpenCV DNN (res10_300x300_ssd_iter_140000.caffemodel)
       в”Ӯ     вҶ’ detect faces вҶ’ collect face bounding boxes
       в”Ӯ     Failure mode: model file missing вҶ’ log warning, skip to Stage 3
       в”Ӯ
       в”ңв”Җв”Җв–¶ [Stage 3: Logo Detection]
       в”Ӯ     OpenCV matchTemplate (multi-scale: 0.5x to 2.0x in 0.1 steps)
       в”Ӯ     вҶ’ for each logo template in config вҶ’ collect match regions
       в”Ӯ     Failure mode: no templates configured вҶ’ skip silently
       в”Ӯ     Known limitation: no rotation handling (v2 scope)
       в”Ӯ
       в””в”Җв”Җв–¶ [Merge & Render]
             Collect all detected regions as individual bounding boxes
             Merge ONLY overlapping boxes (IoU > 0.3) into single boxes
             Non-overlapping regions are redacted independently
             Reversible mode: Gaussian blur each region (kernel=51)
             Irreversible mode: solid black fill each region
             вҶ’ output: new image file at output path (same format as input)
```

## Implementation Approach

Selected approach: **OCR + CV Hybrid** for images, **PyInstaller --onedir + Inno Setup + GitHub Actions** for distribution.

### Phase 1: Core Engine Extensions
**Exit criteria:** All new engine features have passing tests; `anonymizer.anonymize_file()` handles both text and image files; irreversible mode produces output with no mapping file.

1. Irreversible mode flag in Anonymizer class
2. Image pipeline module: `image_anonymizer.py` with three independent stages
3. Image parsers: add `.jpg`, `.jpeg`, `.png`, `.bmp`, `.tiff` to supported file types
4. Image output: blur/black-box regions, save to temp path

### Phase 2: GUI + Distribution
**Exit criteria:** GUI launches on Windows, processes files end-to-end, produces correct output; installer created by CI/CD; Lite build ~150-250MB (hard limit 300MB).

5. tkinter GUI: main window, file picker/drag, mode selector, progress bar
6. Before/After preview panel (800px max width, single file)
7. Batch processing wrapper (folder recursive scan + structured output)
8. Config Manager: export/import .zip with schema validation
9. Auto-update check module (GitHub Releases API, 3s timeout, passive)
10. PyInstaller spec file (--onedir, bundled Tesseract + OpenCV DNN model)
11. Inno Setup script (creates installer from --onedir output)
12. GitHub Actions workflow (Windows runner, two build targets: Full + Lite)

### Key Technical Decisions
- **OCR engine:** pytesseract (direct Tesseract wrapper) + existing custom/regex/NER detectors вҖ” NOT Presidio Image Redactor (Presidio's PII detectors are English-focused and don't recognize Taiwanese school names/IDs; pytesseract uses the same Tesseract OCR engine but feeds output to our Taiwan-optimized detectors); on Windows, bundled as portable binary inside --onedir
- **Irreversible mode:** MappingManager handles label generation вҖ” `get_or_create_token()` returns `[CATEGORY]` label instead of token when `reversible=False`, and skips storing in mapping dict. `_apply_spans` unchanged.
- **Batch processing:** Single-threaded sequential with progress bar (sufficient for 50 files in 2-7 min; multi-process deferred to avoid PyInstaller `freeze_support()` complexity)
- **Face detection:** OpenCV DNN with `res10_300x300_ssd_iter_140000.caffemodel` (~10MB) вҖ” CPU-only, no GPU needed; chosen over YOLO for smaller model size and simpler integration
- **Logo detection:** OpenCV `matchTemplate` with multi-scale search; no rotation handling in v2 (known limitation, acceptable)
- **GUI framework:** tkinter (built into Python, zero extra dependency for PyInstaller)
- **Build format:** PyInstaller `--onedir` (required because Tesseract binary must be co-located); wrapped in Inno Setup installer for clean UX (Start Menu shortcut, uninstaller, etc.)
- **Two build targets:** Full (with NER/PyTorch, ~2-3GB installer) and Lite (without NER, ~150-250MB installer, pending actual measurement)
- **Config schema:** Versioned (`"version": 1`); import validates version and rejects incompatible configs with user-facing error message
